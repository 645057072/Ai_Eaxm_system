# 考试系统项目详设

## 1. 概述

本系统为「生成式 AI 工程师（高级）理论模拟」类在线考试**全栈**：**统一前端**（Vue 3 + Vite + Element Plus + Pinia + Vue Router，**Hash 路由**，管理端与考生端同一 SPA，按 **JWT 角色** 显示菜单）；后端 **Python 3.11+、FastAPI、Uvicorn、SQLAlchemy 2.x、Alembic**；数据库 **MySQL 8.0**；**Redis** 配置预留；容器 **Docker + Nginx** 同域反代 `/api`。  
本文档与实现同步维护：**功能菜单 / 表单 / 列表** 与 **数据库表**、**REST API** 一一对应，减少前后端返工。

**OpenAPI 文档**：服务启动后访问 **`/docs`**（Swagger UI）、**`/redoc`**（经 Nginx 反代，与前端同端口 **80**）。

## 3. 功能菜单与页面（前后端对齐）

以下按 **菜单 → 典型页面 → 主要 API** 列出，便于前端路由与表单字段对齐。

**统一前端路由（Hash）与角色**：`/login` 登录；`/` 首页；`/users`（admin）；`/questions`、`/papers`、`/papers/:id`、`/sessions`（admin、teacher）；`/exam/available`、`/exam/take/:sessionId`（student）；`/attempts/:id` 成绩/答卷（登录用户，后端再校验数据权限）。

### 3.1 认证与个人信息

| 菜单/页面 | 类型 | 说明 | 主要 API |
|-----------|------|------|----------|
| 登录 | 表单 | 用户名、密码 | `POST /api/v1/auth/login` |
| 个人信息 | 展示 | 当前用户、角色 | `GET /api/v1/auth/me` |

### 3.2 用户管理（角色：admin）

| 菜单/页面 | 类型 | 说明 | 主要 API |
|-----------|------|------|----------|
| 用户列表 | 列表+分页 | 支持关键词（用户名模糊） | `GET /api/v1/users` |
| 新建用户 | 表单 | 用户名、密码、姓名、角色 | `POST /api/v1/users` |
| 编辑用户 | 表单 | 姓名、角色、启用、改密 | `PATCH /api/v1/users/{user_id}` |
| 删除用户 | 操作 | 不可删当前登录用户 | `DELETE /api/v1/users/{user_id}` |

### 3.3 题库管理（角色：admin、teacher）

| 菜单/页面 | 类型 | 说明 | 主要 API |
|-----------|------|------|----------|
| 题目列表 | 列表+分页 | 筛选题型、状态 | `GET /api/v1/questions` |
| 题目详情/编辑 | 表单 | 题干、题型、选项 JSON、答案 JSON、解析、难度、状态 | `GET/POST/PATCH/DELETE /api/v1/questions` 及 `/{id}` |

**题型编码 `q_type`**：`judge`（判断）、`single`（单选）、`multiple`（多选）、`fill`（填空）。  
**答案 JSON 约定**：判断用布尔或可解析为「正确/错误」的字符串；单选为选项键如 `"A"`；多选为数组如 `["A","C"]`；填空为字符串。

### 3.4 试卷管理（角色：admin、teacher）

| 菜单/页面 | 类型 | 说明 | 主要 API |
|-----------|------|------|----------|
| 试卷列表 | 列表+分页 | 无小题明细 | `GET /api/v1/papers` |
| 试卷详情/编辑 | 表单+子表 | 标题、说明、时长；小题列表、排序、分值 | `GET/PATCH/DELETE /api/v1/papers/{id}` |
| 向试卷加题 | 表单 | 题目 ID、排序、分值 | `POST /api/v1/papers/{id}/items` |
| 移除小题 | 操作 |  | `DELETE /api/v1/papers/{id}/items/{item_id}` |

### 3.5 考试场次（角色：admin、teacher；考生只读部分接口）

| 菜单/页面 | 类型 | 说明 | 主要 API |
|-----------|------|------|----------|
| 场次列表 | 列表+分页 | 按状态筛选 | `GET /api/v1/exam-sessions` |
| 新建/编辑场次 | 表单 | 绑定试卷、标题、开始/结束时间 | `POST/PATCH /api/v1/exam-sessions` |
| 发布场次 | 操作 | 状态改为 `published` | `POST /api/v1/exam-sessions/{id}/publish` |
| 考生可参加列表 | 列表 | 已发布且在时间窗内 | `GET /api/v1/exam-sessions/available/list` |
| 考试题目（无答案） | 只读 | 题干与选项，不含标准答案 | `GET /api/v1/exam-sessions/{id}/take-data` |
| 开始考试 | 操作 | 生成唯一作答记录 | `POST /api/v1/exam-sessions/{id}/start` |

### 3.6 考试作答（角色：student）

| 菜单/页面 | 类型 | 说明 | 主要 API |
|-----------|------|------|----------|
| 答题/保存 | 表单 | 批量提交各题答案 JSON | `PUT /api/v1/attempts/{attempt_id}/answers` |
| 交卷 | 操作 | 客观题自动阅卷、汇总总分 | `POST /api/v1/attempts/{attempt_id}/submit` |
| 答卷查看 | 展示 | 考生仅本人；教师/管理员可查 | `GET /api/v1/attempts/{attempt_id}` |

## 4. 数据库表结构

字符集：**utf8mb4**。引擎：**InnoDB**。时间字段带时区由应用层写入 UTC；MySQL 存 `DATETIME(6)`。

### 4.1 `sys_role`（角色）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK AI | 主键 |
| name | VARCHAR(64) | 展示名称 |
| code | VARCHAR(32) UNIQUE | `admin` / `teacher` / `student` |
| description | VARCHAR(255) | 说明 |
| created_at | DATETIME(6) | 创建时间 |

### 4.2 `sys_user`（用户）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK AI | 主键 |
| username | VARCHAR(64) UNIQUE | 登录名 |
| password_hash | VARCHAR(255) | 密码哈希 |
| full_name | VARCHAR(64) | 姓名 |
| role_id | BIGINT FK → sys_role.id | 角色 |
| is_active | BOOL | 是否启用 |
| created_at / updated_at | DATETIME(6) | 时间戳 |

### 4.3 `qb_question`（题目）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK AI | 主键 |
| q_type | VARCHAR(16) | 题型 |
| stem | TEXT | 题干 |
| options_json | JSON | 选项（可为 null，如判断题） |
| answer_json | JSON | 标准答案 |
| analysis | TEXT | 解析 |
| difficulty | INT | 1–5 |
| status | VARCHAR(16) | `draft` / `published` |
| created_by | BIGINT FK → sys_user.id | 创建人 |
| created_at / updated_at | DATETIME(6) | 时间戳 |

### 4.4 `exam_paper`（试卷）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK AI | 主键 |
| title | VARCHAR(200) | 标题 |
| description | TEXT | 说明 |
| duration_minutes | INT | 考试时长（分钟） |
| total_score | DECIMAL(10,2) | 小题分值合计（维护） |
| created_by | BIGINT FK → sys_user.id | 创建人 |
| created_at / updated_at | DATETIME(6) | 时间戳 |

### 4.5 `exam_paper_item`（试卷题目项）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK AI | 主键 |
| paper_id | BIGINT FK → exam_paper.id ON DELETE CASCADE | 试卷 |
| question_id | BIGINT FK → qb_question.id ON DELETE CASCADE | 题目 |
| sort_order | INT | 排序 |
| score | DECIMAL(10,2) | 本题分值 |
| 约束 | UNIQUE(paper_id, question_id) | 同卷同题唯一 |

### 4.6 `exam_session`（考试场次）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK AI | 主键 |
| paper_id | BIGINT FK → exam_paper.id ON DELETE CASCADE | 试卷 |
| title | VARCHAR(200) | 场次标题 |
| start_at / end_at | DATETIME(6) NULL | 允许进入与答题的时间窗 |
| status | VARCHAR(16) | `draft` / `published` / `closed` |
| created_by | BIGINT FK → sys_user.id | 创建人 |
| created_at / updated_at | DATETIME(6) | 时间戳 |

### 4.7 `exam_attempt`（作答实例）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK AI | 主键 |
| session_id | BIGINT FK → exam_session.id ON DELETE CASCADE | 场次 |
| user_id | BIGINT FK → sys_user.id ON DELETE CASCADE | 考生 |
| started_at | DATETIME(6) | 开始时间 |
| submitted_at | DATETIME(6) NULL | 交卷时间 |
| status | VARCHAR(16) | `in_progress` / `submitted` / `timeout` |
| total_score | DECIMAL(10,2) NULL | 总分 |
| 约束 | UNIQUE(session_id, user_id) | 每人每场一次 |

### 4.8 `exam_answer`（单题作答）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK AI | 主键 |
| attempt_id | BIGINT FK → exam_attempt.id ON DELETE CASCADE | 作答 |
| question_id | BIGINT FK → qb_question.id ON DELETE CASCADE | 题目 |
| user_answer_json | JSON | 考生答案 |
| score_awarded | DECIMAL(10,2) NULL | 得分 |
| created_at / updated_at | DATETIME(6) | 时间戳 |
| 约束 | UNIQUE(attempt_id, question_id) | 同次同题一条 |

## 6. 初始化与部署

### 6.1 环境变量（节选）

| 变量 | 说明 |
|------|------|
| `DATABASE_URL` | 如 `mysql+pymysql://user:pass@host:3306/exam_db?charset=utf8mb4`（**阿里云 RDS 填内网地址**） |
| `SECRET_KEY` | JWT 密钥（生产须更换） |
| `REDIS_URL` | 预留，如 `redis://host:6379/0` |
| `CORS_ORIGINS` | 逗号分隔；示例含 `http://47.93.44.247` 与本地开发地址；全 `*` 时后端不携带 credentials |

模板见 **`deploy/env.aliyun.example`**，可复制为项目根目录 **`.env`**，由 `docker compose` 自动加载（与 compose 中 `${VAR:-默认值}` 配合）。

### 6.2 本地开发（前后端分离调试）

- **后端**：`pip install -r requirements.txt`，配置 `DATABASE_URL`，执行 `alembic upgrade head` 与 `python scripts/seed_data.py`，`uvicorn app.main:app --reload --port 8000`。  
- **前端**：`cd frontend && npm install && npm run dev`（Vite 默认 **5173**，已配置将 **`/api` 代理到 8000**，与生产同路径）。**Windows** 下若直接执行 `vite` 不在 PATH，`package.json` 已改为通过 **`node ./node_modules/vite/bin/vite.js`** 调用 `build`/`dev`，避免「不是内部或外部命令」。

### 6.3 数据库迁移与种子数据（非 Docker 时）

```bash
alembic upgrade head
python scripts/seed_data.py
```

默认种子：**角色** admin / teacher / student；演示账号（不存在则创建）：

| 用户名 | 初始密码 | 角色 |
|--------|----------|------|
| admin | Admin@123456 | 管理员 |
| teacher | Teacher@123456 | 教师 |
| student | Student@123456 | 考生 |

### 6.4 Docker Compose（推荐：库 + 缓存 + API + 统一 Web）

```bash
docker compose up -d --build
```

- **api** 容器入口脚本会执行 **Alembic 迁移 + 种子** 后启动 Uvicorn；健康检查使用 **`scripts/docker_healthcheck.py`**（8s 超时），`start_period` 300s，适配 ECS 磁盘较慢。  
- **web** 仅依赖 **api 已启动**（`service_started`，不要求 `service_healthy`），避免迁移慢时 **api 长期 unhealthy** 导致 **web 无法启动**、报错 `dependency failed to start: ... api ... is unhealthy`。启动后若 API 尚未就绪，可能出现短暂 502，稍后刷新。  
- **web** 镜像多阶段构建 **前端 `npm run build`**，再由 Nginx 提供 **`/` 静态资源**，并把 **`/api` → api:8000**；**`/health`、`/docs` 等** 见 `deploy/nginx.combined.conf`。  
- 宿主机访问 **`http://<ECS-IP>/`**，前端路由为 **`http://<ECS-IP>/#/login`**（Hash）。

### 6.5 阿里云 ECS（示例：47.93.44.247，Alibaba Cloud Linux 3.2104 LTS 64 位）自动部署

1. **安全组**：放行 **22**、**80**（若上 HTTPS 再开放 443 并配置证书）。  
2. **代码**：将本仓库置于 ECS，例如 **`/opt/exam-system`**（`git clone` 或 CI 同步）。  
3. **环境**：复制 **`deploy/env.aliyun.example`** 为 **`.env`**，填写 **`SECRET_KEY`**、**`CORS_ORIGINS`**（含 `http://47.93.44.247`）；若使用 **RDS**，设置 **`DATABASE_URL`** 为 RDS **内网**连接串，并在 RDS 白名单/安全组中允许 **本 ECS 访问 3306**。  
4. **首次安装 Docker**（在 ECS 上执行一次）：

```bash
chmod +x deploy/aliyun/01-install-docker-alinux3.sh
./deploy/aliyun/01-install-docker-alinux3.sh
```

重新登录 SSH 后执行 `docker ps` 验证。

5. **发布 / 更新**：

```bash
export DEPLOY_ROOT=/opt/exam-system
chmod +x deploy/aliyun/02-deploy.sh
./deploy/aliyun/02-deploy.sh
```

脚本在 `$DEPLOY_ROOT` 下 `git pull` 后 **`docker compose build` + `up -d`**。浏览器访问：**`http://47.93.44.247/#/login`**。

### 6.6 GitHub 推送一键部署（Actions + SSH）

仓库：**`git@github.com:645057072/Ai_Eaxm_system.git`**。生产 ECS：**`47.93.44.247`**，代码目录：**`/opt/exam-system`**（与 `deploy/aliyun/ecs-first-setup.sh`、`remote-deploy.sh` 一致）。

**一次性：在阿里云 ECS 上准备**

1. 安全组放行 **22**（SSH）。若使用 GitHub 托管 Runner 通过 SSH 部署，需允许 **GitHub Actions 出口访问 ECS:22**（可将 [api.github.com/meta](https://api.github.com/meta) 中 `actions` 网段加入安全组，或短期测试放行来源；详见阿里云与 GitHub 文档）。  
2. 配置 **GitHub SSH 拉代码**：在 ECS 部署用户下生成密钥 `ssh-keygen -t ed25519`，将 **公钥** 加到 GitHub 仓库 **Settings → Deploy keys**（只读即可）或账户 SSH Keys；确保 `git clone git@github.com:645057072/Ai_Eaxm_system.git` 成功。  
3. 将 **公钥** 写入 `~/.ssh/authorized_keys` 中 **与 GitHub Actions 使用的私钥对应** 的另一对密钥（用于 CI 登录 ECS），或使用同一密钥对（不推荐，建议 CI 专用一对）。  
4. 首次安装与克隆（在 ECS 上执行；需已配置好访问 GitHub 的 SSH 公钥）：

```bash
sudo mkdir -p /opt/exam-system && sudo chown "$USER" /opt/exam-system
git clone git@github.com:645057072/Ai_Eaxm_system.git /opt/exam-system
cd /opt/exam-system
bash deploy/aliyun/ecs-first-setup.sh
# 按提示编辑 /opt/exam-system/.env 后：
bash deploy/aliyun/remote-deploy.sh
```

说明：`ecs-first-setup.sh` 会安装 Git、Docker，并复制 `.env` 模板；若目录下已有 `.git`（如上先 `clone` 再执行），则不会重复克隆。

**GitHub 仓库 Secrets（Settings → Secrets and variables → Actions）**

| Name | 说明 |
|------|------|
| `ALIYUN_SSH_USER` | ECS 登录用户名（如 `root`） |
| `ALIYUN_SSH_PRIVATE_KEY` | 能 SSH 登录该 ECS 的**私钥**全文（`-----BEGIN ... PRIVATE KEY-----` 至 `-----END ...`） |

**自动部署失败排查（Actions 报 `ssh: no key found` / `ParsePrivateKey`）**

1. **名称必须一致**：仓库 **Settings → Secrets and variables → Actions** 里新建的是 **Repository secrets**，名称须严格为 **`ALIYUN_SSH_USER`**、**`ALIYUN_SSH_PRIVATE_KEY`**（勿写成 `ALIYUN_PRIVATE_KEY` 等）。  
2. **私钥内容**：粘贴 **完整** 私钥，第一行以 `-----BEGIN` 开头，最后一行以 `-----END` 结尾；勿加引号、勿只粘贴中间一段。  
3. **格式**：须为 **OpenSSH / PEM**（Linux `~/.ssh/id_rsa`、`id_ed25519` 内容）。**PuTTY 的 `.ppk` 不行**，需用 `PuTTYgen` 导出为 OpenSSH 或重新生成一对密钥。  
4. **私钥带密码**：若生成密钥时设置了 passphrase，需在 workflow 的 `appleboy/ssh-action` 中增加 `passphrase: ${{ secrets.xxx }}` 并在仓库中配置对应 Secret。  
5. **ECS 侧**：该私钥对应的 **公钥** 必须在 ECS 的 `~/.ssh/authorized_keys`（与 `ALIYUN_SSH_USER` 家目录一致），且安全组放行 **22**。  

工作流已增加 **「校验 SSH Secrets 已配置」** 步骤：若私钥为空或缺少 `BEGIN` 行，会在连接前失败并给出中文提示。

工作流文件：`.github/workflows/deploy-aliyun-ecs.yml`。向 **`main`** 分支 **`git push`** 后自动执行远程 `deploy/aliyun/remote-deploy.sh`（`git fetch` + `reset` 到 `origin/main` + `docker compose up -d --build`）。也可在 Actions 里 **手动 Run workflow**。

**本机 Windows 推送脚本（可选）**：`scripts/git-push-deploy.ps1`，执行后打开 Actions 页面查看进度。
